## State

**Store**: `character-encounter-statestore` (Backend: MySQL)

All 10 typed store variables share a single `StateStoreDefinitions.CharacterEncounter` definition. Key prefixes distinguish logical namespaces.

| Key Pattern | Data Type | Purpose |
|-------------|-----------|---------|
| `type-{CODE}` | `EncounterTypeData` | Encounter type definitions (built-in and custom) |
| `enc-{encounterId}` | `EncounterData` | Shared encounter records |
| `pers-{perspectiveId}` | `PerspectiveData` | Per-participant perspective records |
| `char-idx-{characterId}` | `CharacterIndexData` | List of perspective IDs for a character |
| `pair-idx-{charA}:{charB}` | `PairIndexData` | List of encounter IDs between a character pair (sorted GUIDs) |
| `loc-idx-{locationId}` | `LocationIndexData` | List of encounter IDs at a location |
| `global-char-idx` | `GlobalCharacterIndexData` | All character IDs with encounter data |
| `custom-type-idx` | `CustomTypeIndexData` | All custom encounter type codes |
| `type-enc-idx-{CODE}` | `TypeEncounterIndexData` | List of encounter IDs using a type code |
| `enc-pers-idx-{encounterId}` | `EncounterPerspectiveIndexData` | List of perspective IDs for an encounter |

### RecordEncounter
POST /character-encounter/record | Roles: []

```
IF body.ParticipantIds.Count < 2                                   -> 400
IF body.ParticipantIds.Count > config.MaxParticipantsPerEncounter   -> 400
// Deduplicate participant IDs
READ _encounterTypeStore:type-{body.EncounterTypeCode.ToUpperInvariant()}
  // auto-seed built-in if needed (see helper: SeedBuiltInTypeAsync)
IF type not found                                                   -> 400
IF type.IsDeprecated                                                -> 400 (Category B guard)
IF !type.IsActive                                                   -> 400
// Duplicate detection: check first pair index for encounters with matching type+timestamp
READ _pairIndexStore:pair-idx-{pairKey}
FOREACH candidateId in pairIndex
  READ _encounterStore:enc-{candidateId}
  IF same type AND timestamp within config.DuplicateTimestampToleranceMinutes -> 409

// Validate all participants exist
FOREACH participant (parallel via Task.WhenAll)
  CALL _characterClient.GetCharacterAsync({ characterId })          -> 404 if any not found

// Create encounter
WRITE _encounterStore:enc-{encounterId} <- new EncounterData
ETAG-WRITE _typeEncounterIndexStore:type-enc-idx-{typeCode}        // add encounterId

// Create perspectives
FOREACH participant
  WRITE _perspectiveStore:pers-{perspectiveId} <- new PerspectiveData
  // Default emotional impact from type or outcome; default sentiment from outcome config
  ETAG-WRITE _characterIndexStore:char-idx-{characterId}
  ETAG-WRITE _globalCharacterIndexStore:global-char-idx             // add characterId on first perspective
  ETAG-WRITE _encounterPerspectiveIndexStore:enc-pers-idx-{encounterId}

// Create pair indexes (O(N^2) pairs)
FOREACH unique pair
  ETAG-WRITE _pairIndexStore:pair-idx-{charA}:{charB}

IF body.LocationId != null
  ETAG-WRITE _locationIndexStore:loc-idx-{locationId}

// Prune if limits exceeded
// see helper: PruneCharacterEncountersIfNeededAsync, PrunePairEncountersIfNeededAsync

PUBLISH encounter.recorded { encounterId, encounterTypeCode, outcome, realmId, locationId, participantIds, context, encounterTimestamp }

// Register character references with lib-resource
FOREACH participant
  CALL _resourceClient.RegisterReferenceAsync(...)

RETURN (200, EncounterResponse { encounter, perspectives })
```

  READ _perspectiveStore:pers-{perspectiveId}

  // Collect unique encounter IDs
FOREACH unique encounterId
  READ _encounterStore:enc-{encounterId}

  READ _encounterPerspectiveIndexStore:enc-pers-idx-{encounterId}

  ETAG-WRITE _typeEncounterIndexStore:type-enc-idx-{typeCode}

### DecayMemories
POST /character-encounter/decay-memories | Roles: []

```
IF !config.MemoryDecayEnabled                                      -> 200 (processed=0, faded=0)
IF body.CharacterId != null
  // Target single character
ELSE
  READ _globalCharacterIndexStore:global-char-idx
  // Target all characters
FOREACH characterId
  READ _characterIndexStore:char-idx-{characterId}
  FOREACH perspectiveId
    READ _perspectiveStore:pers-{perspectiveId} [with ETag]
    // Calculate decay: intervals = hours since lastDecayed / config.MemoryDecayIntervalHours
    // decayAmount = intervals * config.MemoryDecayRate
    IF intervals < 1: skip
    IF body.DryRun: count only, no write
    ELSE
      ETAG-WRITE _perspectiveStore:pers-{perspectiveId}            // skip on ETag conflict (Warning)
      IF crossed below config.MemoryFadeThreshold
        PUBLISH encounter.memory.faded { encounterId, characterId, perspectiveId, previousStrength, newStrength, fadeThreshold }
RETURN (200, DecayMemoriesResponse { perspectivesProcessed, memoriesFaded })
```

### RestoreFromArchive
POST /character-encounter/restore-from-archive | Roles: []

```
// Decompress Base64-encoded GZip JSON → CharacterEncounterArchive
IF deserialization fails                                           -> 400
FOREACH encounter in archive
  READ _encounterStore:enc-{encounterId}
  IF exists: skip (idempotent)
  WRITE _encounterStore:enc-{encounterId} <- restored EncounterData
  ETAG-WRITE _pairIndexStore:pair-idx-{pairKey}                   // for each pair
  IF locationId: ETAG-WRITE _locationIndexStore:loc-idx-{locationId}

  ETAG-WRITE _characterIndexStore:char-idx-{characterId}
  ETAG-WRITE _encounterPerspectiveIndexStore:enc-pers-idx-{encounterId}

RETURN (200, RestoreFromArchiveResponse { encountersRestored, perspectivesRestored })
```

## Background Services

### MemoryDecaySchedulerService
**Interval**: `config.ScheduledDecayCheckIntervalMinutes` (default: 60 minutes)
**Startup Delay**: `config.ScheduledDecayStartupDelaySeconds` (default: 30 seconds)
**Purpose**: Scheduled memory decay processing (only active when `MemoryDecayEnabled=true` AND `MemoryDecayMode=Scheduled`)

```
// Self-disables if config.MemoryDecayEnabled=false OR config.MemoryDecayMode != Scheduled
// Follows canonical worker polling loop skeleton
// Each cycle:
//   When DecayTimeSource == GameTime:
//     For each perspective, resolve encounter's realmId, call IWorldstateClient.GetElapsedGameTimeAsync
//     Use game-seconds for decay interval calculation instead of real-time hours
//   When DecayTimeSource == RealTime:
//     Use existing real-time calculation (hours since last decay)
CALL CharacterEncounterService.DecayMemoriesAsync({ characterId: null, dryRun: false })
  // Delegates to the service method which handles global character index iteration
// Per-cycle telemetry span; per-item error isolation; WorkerErrorPublisher on cycle failure
```

## Non-Standard Implementation Patterns

#### Plugin Lifecycle (OnRunningAsync)

```
// Step 1: Register character cleanup callbacks with lib-resource
CALL CharacterEncounterReferenceTracking.RegisterAsync(resourceClient)
  // Registers CASCADE cleanup at /character-encounter/delete-by-character

// Step 2: Register compression callbacks for character archival
CALL CharacterEncounterCompressionCallbacks.RegisterAsync(resourceClient)
  // Priority 30, compress via /character-encounter/get-compress-data, decompress via /character-encounter/restore-from-archive

// Step 3: Register ABML event templates (soft dependency)
IF IEventTemplateRegistry available
  CALL CharacterEncounterEventTemplates.RegisterAll(eventTemplateRegistry)
  // Templates: encounter_recorded, encounter_memory_faded
```

Both lib-resource registrations (steps 1-2) log Warning on failure without crashing. Step 3 is fully soft.

#### ConfigureServices

Registers `MemoryDecaySchedulerService` as a hosted service. The scheduler self-disables at startup if `MemoryDecayEnabled=false` OR `MemoryDecayMode != Scheduled`.
